# The Agenthoryx Foundation Charter

## Mission
The Agenthoryx Foundation is a non-profit organization dedicated to the stewardship, evolution, and standardization of the Agenthoryx programming language and its AI-native ecosystem. We ensure that Agenthoryx remains an open, accessible, and globally recognized standard for intelligent software development.

## Core Responsibilities
1. **Language Governance**: Overseeing the evolution of the Agenthoryx Language Specification.
2. **Standards Management**: Maintaining the official reference documents for the VM, Bytecode, and Agent runtimes.
3. **Release Management**: Coordinating stable releases and managing the official package registry (`hub.agenthoryx.dev`).
4. **Community Leadership**: Fostering an inclusive open-source community through events, RFCs, and governance.
5. **Research Coordination**: Bridging the gap between academic AI research and industry adoption.

## Organizational Structure

### Foundation Board
The executive body responsible for legal, financial, and strategic decision-making.

### Committees
- **Language Committee**: Evaluates and approves changes to the language syntax and standard library.
- **Research Committee**: Partners with universities to explore advancements in agent systems and distributed intelligence.
- **Security Committee**: Manages vulnerabilities, CVE assignments, and enterprise RBAC standards.
- **Education Committee**: Develops academic curricula, lab manuals, and the official certification program.
- **Ecosystem Committee**: Maintains the Agenthoryx Package Manager (CPM), VS Code extensions, and standard tooling.

## The CEP Process (Agenthoryx Enhancement Proposal)
Any major change to the language or ecosystem must undergo the CEP process:
1. **Draft**: Author submits a proposal to the community.
2. **Review**: The Language Committee and community debate the technical merits.
3. **Accepted/Rejected**: The Committee issues a formal decision.
4. **Implementation**: Accepted CEPs are merged into the official specification and reference implementations.
